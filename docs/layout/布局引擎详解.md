# WinElement 布局引擎详解

> 对应源码：`include/winelement/layout/` + `src/layout/`
> 核心组件：LayoutElement、LayoutEngine、GridLayout、PixelSnap

---

## 一、整体架构

### 1.1 设计目标

布局引擎解决的核心问题：**给定一组控件和约束条件，计算出每个控件的位置和大小。**

```
输入                              输出
─────                             ────
LayoutConstraints                  frame() = {x, y, width, height}
  ├─ width: optional<float>        margin() = {l, t, r, b}
  ├─ height: optional<float>       padding() = {l, t, r, b}
  └─ direction: LTR/RTL           border() = {l, t, r, b}
```

### 1.2 由下至上的模块依赖

```
UIElement（逻辑树）
   │  owns
   ▼
LayoutElement（Yoga 节点封装）
   ├─ 44+ set_* 方法（fluent API）
   ├─ 惰性 Yoga 节点创建（ensure_impl）
   ├─ 延迟样式刷新（StyleState）
   └─ 线程本地对象池
       │
       ▼
Yoga（Meta 的 Flexbox 引擎）
   └─ 被 RAII 封装在 detail/yoga_handles.hpp
       ├─ YogaConfigHandle（shared_ptr 管理）
       └─ YogaNodeHandle（unique_ptr 语义）
```

### 1.3 和 UIElement 的分工

| 谁负责 | 做什么 |
|--------|--------|
| **LayoutElement** | 纯布局：计算 frame/margin/padding/border，管理 Yoga 节点 |
| **UIElement** | 更高层：调用 `calculate_layout`，同步 frame 到 `committed_frame_`，处理滚动/虚拟化/溢出 |

LayoutElement 不知道事件、不知道绘制、不知道样式——它只做一件事：给定约束，算出位置。

## 二、LayoutElement（布局节点）

### 2.1 内存布局

```
LayoutElement (sizeof ≈ 160-200 bytes)
├── vtable_ptr (8 bytes, 虽然 final 但有虚函数？不——LayoutElement 是 final，但继承自... 实际上它没有虚函数)
│
├── children_: vector<unique_ptr<LayoutElement>>    // 子节点列表
├── parent_: LayoutElement*                         // 父指针（树遍历用）
│
├── config_: shared_ptr<YogaConfigHandle>           // Yoga 配置
├── impl_: unique_ptr<Impl>                         // Yoga 节点（惰性创建）
│
├── measure_callback_: MeasureCallback              // std::function<Size(MeasureInput)>
├── baseline_callback_: BaselineCallback
├── dirtied_callback_: DirtiedCallback
│
├── style_state_: unique_ptr<StyleState>            // 延迟样式存储（惰性创建）
│
├── cached_frame_: Rect                             // 上次布局结果缓存
├── cached_layout_direction_: Direction
├── cached_layout_margin_: EdgeInsets
├── cached_layout_padding_: EdgeInsets
├── cached_layout_border_: EdgeInsets
├── cached_has_new_layout_: bool
├── cached_had_overflow_: bool
│
└── measure_dirty_pending_: bool
```

### 2.2 Impl（Yoga 节点封装）

```cpp
class Impl {
    shared_ptr<YogaConfigHandle> config_;  // 共享配置
    YogaNodeHandle node_;                  // RAII 包装的 YGNodeRef
    LayoutElementContext context_;         // {owner, measure*, baseline*, dirtied*}
};
```

`YogaNodeHandle` 构造时调用 `YGNodeNewWithConfig`，析构时调用 `YGNodeFree`。Move-only。

`LayoutElementContext` 存放在 `YGNodeSetContext` 中，在 Yoga C 回调函数中通过 `YGNodeGetContext` 取回。

### 2.3 StyleState（延迟样式存储）

LayoutElement 的所有 setter（`set_width`、`set_flex`、`set_margin` 等）**不直接调 Yoga API**，而是写入 `StyleState` 中的 `std::optional<T>`：

```cpp
struct StyleState {
    std::optional<Length> width_;
    std::optional<Length> height_;
    std::optional<Length> min_width_;
    std::optional<Length> min_height_;
    std::optional<Length> max_width_;
    std::optional<Length> max_height_;
    std::optional<FlexDirection> flex_direction_;
    std::optional<JustifyContent> justify_content_;
    // ... 共 ~28 个 optional 字段
};
```

**为什么这么做？**

本质原因是 Yoga 的 C API 不允许"取消设置"一个属性——一旦你调了 `YGNodeStyleSetWidth`，这个值就在 Yoga 里永久存在了。而 C++ 端不一定每帧都设置每个属性。

**只有当 `ensure_impl()` 被调用时**，`apply_pending_style_to_impl()` 才会遍历所有 optional 字段，把有值的刷新到 Yoga：

```
set_width(100)  →  style_state_->width_ = 100（optional）
set_flex(1)     →  style_state_->flex_grow_ = 1（optional）
                   ⋮
calculate_layout() → ensure_impl()
                      → apply_pending_style_to_impl()
                          → YGNodeStyleSetWidth(node, 100)
                          → YGNodeStyleSetFlexGrow(node, 1)
```

### 2.4 子节点管理

```cpp
children_: std::vector<std::unique_ptr<LayoutElement>>
```

| 方法 | 复杂度 | 说明 |
|------|--------|------|
| `child_at(i)` | O(1) | 直接索引 |
| `append_child(c)` | O(1) | push_back，并 YGNodeInsertChild |
| `insert_child(i, c)` | O(n) | 按位置插入，需要 shift |
| `remove_child_at(i)` | O(n) | 按位置移除，需要 shift |
| `clear_children()` | O(n) | 逐个销毁 |

插入/移除时同时维护 LayoutElement 的 `children_` 向量和 Yoga 的节点树。

## 三、LayoutEngine（引擎工厂）

```cpp
class LayoutEngine {
    shared_ptr<YogaConfigHandle> config_;
public:
    auto create_element() -> unique_ptr<LayoutElement>;  // 工厂
    
    // Yoga 配置透传
    void set_point_scale_factor(float);
    void set_errata(Errata);
    void set_experimental_feature_enabled(ExperimentalFeature, bool);
    // ...
};
```

LayoutEngine 本身就是一个 Yoga 配置的包装器。多个 LayoutElement 共享同一个 `config_`（通过 `shared_ptr`），保证同一棵树内的配置一致性。

### 3.1 绑定流程

```cpp
// UIElement 绑定到 LayoutEngine
void UIElement::bind_layout_tree(LayoutEngine& engine) {
    layout_->bind_to_engine(engine);      // 替换 config_
    layout_->materialize_subtree();        // 为所有节点创建 Yoga impl
}
```

`materialize_subtree()` 递归调用 `ensure_impl()`，确保整棵树的 Yoga 节点都被创建并连接好。

### 3.2 切换引擎

```cpp
void LayoutElement::bind_to_engine(LayoutEngine& engine) {
    config_ = engine.config_;  // 替换 shared_ptr
    drop_impl();               // 销毁旧的 Yoga 节点
    // 递归到子节点...
}
```

切换引擎是轻微的代价——所有 Yoga 节点被销毁，下次布局时重建。

## 四、核心流程

### 4.1 约束传递

```
UIElement::calculate_layout(LayoutConstraints)
  │
  ├── LayoutConstraints = {width: 可选, height: 可选}
  │     └── optional 为空时 → YGUndefined（NaN）
  │
  └── LayoutElement::calculate_layout(constraints)
        │
        ├── 1. ensure_impl() — 创建 Yoga 节点 + 刷新样式
        │
        ├── 2. YGNodeCalculateLayout(
        │         node, width_or_NaN, height_or_NaN, direction)
        │      └── Yoga 递归计算整棵子树
        │
        ├── 3. 缓存结果到 cached_frame_ / cached_layout_margin_ 等
        │
        └── 4. UIElement::sync_layout_snapshot_subtree()
              ├── committed_frame_ = layout_->frame()
              ├── committed_absolute_frame_ = 父绝对位置 + 偏移
              └── 递归到子节点
```

### 4.2 约束的可用性检查

```cpp
float available_or_undefined(std::optional<float> value, const char* name) {
    if (!value.has_value())
        return YGUndefined;  // 无约束 → NaN
    
    const auto v = value.value();
    if (!std::isfinite(v) || v < 0.0F)
        throw std::invalid_argument(/* ... */);
    
    return v;
}
```

YGUndefined 在 Yoga 中由 NaN 表示，意味着"不限制"。

### 4.3 失效传播

```
UIElement::set_width(100)
  → layout_element.set_width(100)  // 写入 StyleState
  → UIElement::invalidate_layout()
     ├── needs_layout_ = true
     ├── self_needs_paint_ = true
     ├── needs_paint_ = true
     └── 向上传播, 直到根节点或 relayout_boundary_
          ├── relayout_boundary_ 存在 → mark_descendant_layout_dirty（轻量）
          └── 不存在 → invalidate_layout（递归）
```

**`relayout_boundary_`**：如果一个元素有自己的固定尺寸约束（比如 `position: absolute` 或固定宽高），它的尺寸变化不影响父级的布局——标记为 boundary。失效传播在这里停止。

## 五、Measure Callback

### 5.1 用途

有些元素的大小不是由子节点或固定尺寸决定的，而是由**内容**决定的。比如文本控件——宽度受约束，高度由文本内容决定。这种"根据约束计算内容大小"的机制就是 Measure Callback。

### 5.2 注册

```cpp
LayoutElement& LayoutElement::set_measure_callback(MeasureCallback callback) {
    if (!children_.empty()) {
        throw std::logic_error("子节点和 measure callback 不能共存");
    }
    measure_callback_ = std::move(callback);
    // 设置 Yoga 的 measure 函数指针
    YGNodeSetMeasureFunc(node, measure_layout_element);
    return *this;
}
```

**一个元素要么有子节点，要么有 measure callback，不能同时存在。**

### 5.3 C 回调跳板

```cpp
YSize measure_layout_element(YGNodeConstRef node, float width,
                              YGMeasureMode width_mode, float height,
                              YGMeasureMode height_mode) {
    auto* ctx = static_cast<LayoutElementContext*>(YGNodeGetContext(node));
    if (!ctx || !ctx->measure_callback || !*ctx->measure_callback)
        return {0, 0};  // 安全降级
    
    MeasureInput input{width, from_yoga(width_mode), height, from_yoga(height_mode)};
    Size result = (*ctx->measure_callback)(input);
    return {clamp_to_nonnegative(result.width), clamp_to_nonnegative(result.height)};
}
```

Yoga 回调中不能抛异常，所以所有异常被 catch 并返回 `{0, 0}`。NaN/Infinity 被 clamp 到 0。

### 5.4 流程

```
文本控件内容变了
  → mark_measure_dirty()
    → YGNodeMarkDirty(node)  → Yoga 标记此节点需要重新 measure
    → dirtied_callback_ fires → UIElement::invalidate_layout()
      → 下一帧 calculate_layout()
        → Yoga 再次调用 measure_layout_element
          → 传入可用宽度
          → MeasureCallback 用 DirectWrite 测文本尺寸
          → 返回 {text_width, text_height}
```

## 六、Grid Layout（独立于 Yoga）

Grid 不是用 Yoga 实现的，而是一个独立的计算器。

### 6.1 核心类型

```cpp
struct GridTrack {
    GridTrackSizing sizing;  // Fixed | Star
    float value;             // 固定值或 Star 权重
    float min;               // 最小值
    float max;               // 最大值（默认 inf）
};

struct GridItem {
    size_t row, column;      // 起始位置
    size_t row_span;         // 跨几行（默认 1）
    size_t column_span;      // 跨几列（默认 1）
};
```

### 6.2 轨道解析算法

```
resolve_tracks(tracks, available_extent, gap):
  第一遍：固定轨道
    for each 固定轨道:
      尺寸 = clamp(value, min, max)
      剩余空间 -= (尺寸 + gap)
  
  第二遍：Star 轨道
    remaining = max(0, 剩余空间)
    total_star = sum(star_weights)
    for each star 轨道:
      尺寸 = remaining * (weight / total_star)
      clamp(尺寸, min, max)
```

**复杂度 O(n)**，n = 轨道数。

### 6.3 项目放置

```cpp
arrange(items, rows, columns, gap):
  for each item:
    row_end = min(总行数, row + max(span, 1))
    col_end = min(总列数, column + max(span, 1))
    x = prefix_sum(columns, item.column) + item.column * gap
    y = prefix_sum(rows, item.row) + item.row * gap
    width = prefix_sum(columns, col_end) - x
    height = prefix_sum(rows, row_end) - y
```

`prefix_sum` 每次从头加和，复杂度 O(span)。总计 O(m × s)，m = 项目数，s = 平均跨度。

### 6.4 实际使用

GridLayoutPlanner 不直接操作 LayoutElement。它输出 `GridLayoutResult`（行高、列宽、项目矩形数组），由调用者（CustomLayoutCallback）应用到子节点。

## 七、像素对齐

### 7.1 原理

UI 元素在不同 DPI 屏幕上显示时，如果坐标落在非整数像素上，会产生模糊的边缘。解决方案是把坐标对齐到最近的像素边界。

```
公式：snapped = round(value * scale) / scale
```

scale = 1.0（100% DPI）：value 1.23 → round(1.23) → 1.0
scale = 2.0（200% DPI）：value 1.23 → round(2.46) / 2 → 2.0 / 2 → 1.0

### 7.2 Rect 对齐

```cpp
Rect snap_rect_to_pixels(Rect rect, float scale) {
    float left   = snap_to_pixel(rect.x, scale);
    float top    = snap_to_pixel(rect.y, scale);
    float right  = snap_to_pixel(rect.x + rect.width, scale);
    float bottom = snap_to_pixel(rect.y + rect.height, scale);
    return {left, top, right - left, bottom - top};
}
```

四条边分别对齐，然后重构 width/height。这保证 `left + width == right`，防止相邻元素之间出现亚像素间隙。

### 7.3 生效位置

- `YGConfigSetPointScaleFactor(config, scale)`：Yoga 内部自动对齐
- `snap_rect_to_pixels()`：手动对齐，用于 paint 时的精确像素级定位

## 八、对象池

### 8.1 LayoutElement 线程本地池

LayoutElement 重写了 `operator new`/`delete`，使用**线程本地自由链表**：

```cpp
// 每个线程一个池
static thread_local LayoutElementThreadPool pool;

void* LayoutElement::operator new(size_t size) {
    return pool.allocate(size);  // 优先从池中分配
}

void LayoutElement::operator delete(void* ptr) noexcept {
    pool.deallocate(ptr, sizeof(LayoutElement));  // 归还到池
}
```

- `allocate`：自由链表非空时 O(1) pop，否则 fallback 到 `::operator new`
- `deallocate`：O(1) push 到自由链表头部

### 8.2 LayoutElementPool

更上层的 `unique_ptr` 池，用于频繁创建/销毁的场景（如虚拟化）：

```cpp
class LayoutElementPool {
    LayoutEngine* engine_;
    vector<unique_ptr<LayoutElement>> pool_;
public:
    auto acquire() -> unique_ptr<LayoutElement>;  // 池非空则取，否则新建
    void release(unique_ptr<LayoutElement>);       // reset_for_pool + 放回池
    void clear();                                  // 清空池
};
```

`reset_for_pool()` 清空所有状态（子节点、样式、回调、缓存、impl），绑定到引擎配置，然后推入池中。

## 九、帧缓存策略

LayoutElement 缓存了 7 个布局输出值：

| 字段 | 来自 Yoga 的 API | 无 impl 时的行为 |
|------|------------------|-----------------|
| `cached_frame_` | `YGNodeLayoutGetLeft/Top/Width/Height` | 全 0 |
| `cached_layout_direction_` | `YGNodeLayoutGetDirection` | `Inherit` |
| `cached_layout_margin_` | `YGNodeLayoutGetMargin(4 edges)` | 全 0 |
| `cached_layout_padding_` | `YGNodeLayoutGetPadding(4 edges)` | 全 0 |
| `cached_layout_border_` | `YGNodeLayoutGetBorder(4 edges)` | 全 0 |
| `cached_has_new_layout_` | `YGNodeGetHasNewLayout` | false |
| `cached_had_overflow_` | `YGNodeLayoutGetHadOverflow` | false |

每次 `calculate_layout()` 后更新所有缓存。如果 Yoga 节点被销毁（`drop_impl`），最后一次已知值保留在缓存中——不会突然归零。

UIElement 在此基础上增加两层：

```
LayoutElement::frame()        → 相对于父元素的 frame
UIElement::committed_frame_   → 相对于父元素的 frame（布局后同步）
UIElement::committed_absolute_frame_ → 相对于根元素的 frame
```

绘制和命中测试都使用 `committed_absolute_frame_`，完全不碰 Yoga。

## 十一、复杂度总结

| 操作 | 复杂度 | 说明 |
|------|--------|------|
| 新建 LayoutElement | O(1) 均摊 | 线程本地自由链表池 |
| set_*(value) | O(1) | 只写入 StyleState 的 optional |
| ensure_impl() | O(k) | k = 子节点数；创建 Yoga 节点 + 刷新样式 + 插入子节点 |
| calculate_layout() | O(n) | Yoga flexbox；n = 子树中节点数 |
| Grid resolve_tracks() | O(t) | t = 轨道数；两遍线性扫描 |
| Grid arrange() | O(m·s) | m = 项目数，s = 平均跨度 |
| snap_to_pixel() | O(1) | 一次 round/multiply/divide |
| LayoutElementPool 获取 | O(1) | vector pop_back |
