# 06 — 纹理与图集

> 对应渲染器概念：`glyph_atlas_texture_`、`glyph_atlas_view_`

## 什么是纹理

纹理就是一张可以在 GPU 上采样的图片。D3D11 中用 `ID3D11Texture2D` 表示。

## 创建纹理

```cpp
D3D11_TEXTURE2D_DESC td = {};
td.Width       = 1024;             // 宽
td.Height      = 1024;             // 高
td.Format      = DXGI_FORMAT_B8G8R8A8_UNORM;  // 每像素 4 字节
td.BindFlags   = D3D11_BIND_SHADER_RESOURCE | D3D11_BIND_RENDER_TARGET;
td.Usage       = D3D11_USAGE_DEFAULT;
td.MipLevels   = 1;                // 无 mipmap（UI 不需要）
td.ArraySize   = 1;

ID3D11Texture2D* texture;
device->CreateTexture2D(&td, nullptr, &texture);
```

## 创建着色器资源视图（SRV）

D3D11 不能直接用纹理——要通过一个"视图"来访问它：

```cpp
ID3D11ShaderResourceView* view;
device->CreateShaderResourceView(texture, nullptr, &view);
```

像素着色器绑定时：

```cpp
context->PSSetShaderResources(0, 1, &view);  // 绑定到槽 0
```

在着色器里：

```hlsl
Texture2D tex : register(t0);         // 绑定到纹理槽 0
SamplerState samp : register(s0);     // 绑定到采样器槽 0

float4 main(VSOutput input) : SV_Target {
    return tex.Sample(samp, input.uv);  // 采样纹理
}
```

## 字形图集（Glyph Atlas）

字形图集是 UI 渲染中的一个核心技巧。

### 不如图集的做法

每个文字字形一个单独的纹理：

```
纹理 A (16x16)    纹理 B (16x16)    纹理 C (16x16)    ... 几千个纹理
[  "A"  ]          [  "B"  ]          [  "C"  ]
```

每画一个字就要切换纹理 → 切纹理是 GPU 性能杀手 → 文字一多就卡。

### 用图集的做法

把所有字形排到一张大纹理上：

```
纹理 (1024x1024)
┌────────────┬────────────┬────────────┬───────  ─
│     A      │     B      │     C      │   D
├────────────┼────────────┼────────────┼───────  ─
│     E      │     F      │     G      │   H
├────────────┼────────────┼────────────┼───────  ─
│     I      │     J      │     K      │   L
├────────────┼────────────┼────────────┼───────  ─
│    ...     │            │            │
```

每个字形知道自己在图集中的 UV 坐标（左上角 + 宽高）。画文字时只用一张纹理、一次绑定，画出所有字，**不需要切换纹理**。

### 你的图集工作流

```
draw_text("Hello")
  → DirectWrite 把每个字光栅化到 CPU 内存
  → 在图集中找空位，拷贝进去
  → GPU 通过 glyph_atlas_view_ 采样
  → 像素着色器从图集里读出每个字的像素
  → 输出到屏幕
```

## 更新图集

新字形渲染出来后需要上传到 GPU：

```cpp
// CPU 侧：DirectWrite 渲染了一个新字形
BYTE* glyph_pixels = ...;  // 32x32 的灰度图

// 找到图集中的空位（比如从第 512 行开始）
D3D11_BOX box = {};
box.left   = atlas_x;        // 图集中的 X 位置
box.top    = atlas_y;        // 图集中的 Y 位置
box.right  = atlas_x + 32;
box.bottom = atlas_y + 32;
box.front  = 0;
box.back   = 1;

// 把字形数据更新到图集纹理中
context->UpdateSubresource(glyph_atlas_texture_, 0, &box, 
                           glyph_pixels, 32 * 4, 0);
```

**注意**：过多的 `UpdateSubresource` 会卡。你的渲染器用**脏区域跟踪**来批量更新——只更新图集中变化的部分。

## 你的渲染器中的纹理

| 纹理 | 用途 |
|------|------|
| `glyph_atlas_texture_` | 字形图集，所有文字字形 |
| 用户图片（在 `d3d11_render_resource_cache` 中） | 用户通过 `Image` 控件加载的图片 |

## 为什么要 1024x1024？

更大 = 可容纳更多字形，但占更多显存且图集重置时的卡顿更久。
更小 = 显存省了，但频繁重置导致卡顿。

你的图集当前是 1024x1024，4 字节/像素 = **4MB**。
建议 2048x2048 = **16MB**，可以显著减少重置频率。
