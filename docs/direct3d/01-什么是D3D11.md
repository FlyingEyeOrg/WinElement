# 01 — D3D11 是什么

## GPU vs CPU

你的 UI 框架要做的事情本质上就一件：**把像素填到屏幕上**。

CPU 可以做这件事，但它是"串行"的——一次只能处理一个像素。GPU 是"并行"的——一次可以处理几千个像素。所以画 UI 的渲染工作交给 GPU 更合适。

D3D11 就是微软提供的、让 CPU 给 GPU 下命令的一套接口（API）。

## 保留式 vs 即时模式

理解 D3D11 之前，先分清两种渲染模式：

**即时模式（Immediate Mode）**
```
每帧你告诉 GPU：画一个矩形在这里，画一个圆在那里，画一行字在这里
```
像 ImGui、GDI 都是这样。简单直接，但每帧你的 CPU 都要重新计算所有绘制命令。

**保留式（Retained Mode）**
```
你先建一个"场景"：窗口里有一个按钮，按钮里有一个矩形，矩形里有一行字
然后每帧告诉 GPU：把场景画出来
```
像 WPF、Flutter、你的框架都是这样。更复杂，但 GPU 可以只画变化的部分。

D3D11 本身是**即时模式**的 API——它每帧接收命令并执行。你的框架在你和 D3D11 之间加了一层保留式的抽象（UIElement 树 → RenderCommandList → RenderScene），把保留式逻辑翻译成 D3D11 能懂的即时命令。

## D3D11 的核心设计

D3D11 的架构可以归纳为**三个对象 + 一个管线**：

### 三个对象

| 对象 | 通俗理解 | WinElement 对应 |
|------|----------|-----------------|
| **Device** (`ID3D11Device`) | 显卡工厂——用来创建各种资源（纹理、缓冲区、着色器） | `device_` |
| **Context** (`ID3D11DeviceContext`) | 命令队列——用来下命令（设置状态、画东西） | `primary_deferred_context_` |
| **SwapChain** (`IDXGISwapChain`) | 双缓冲管理器——管理前后缓冲的交换 | swapchain（在 `d3d11_composition_surface.cpp`） |

理解这三个对象的关系是看懂渲染器的起点。

### 一个管线

D3D11 的渲染管线是一个固定流程：

```
顶点数据 → [Input Assembler] → [Vertex Shader] → [Rasterizer] → [Pixel Shader] → [Output Merger] → 屏幕
```

每个阶段：

| 阶段 | 谁执行 | 干什么 |
|------|--------|--------|
| **Input Assembler** | GPU 固定功能 | 从顶点缓冲区读取顶点数据，组装成三角形 |
| **Vertex Shader** | GPU 可编程 | 每个顶点执行一次。把坐标从模型空间转换到屏幕空间 |
| **Rasterizer** | GPU 固定功能 | 把三角形转成像素片段（决定哪些像素被覆盖） |
| **Pixel Shader** | GPU 可编程 | 每个像素执行一次。决定这个像素最终什么颜色 |
| **Output Merger** | GPU 固定功能 | 做深度测试、模板测试、透明度混合 |

你的渲染器做的事情，就是把 UI 控件的绘制需求翻译成"顶点 → 顶点着色器 → 像素着色器"这个流程。

## 从 WinElement 的角度看

```
你写： fill_rect({10, 10, 100, 50}, Color::Red)

框架做了：
  1. 录制到 RenderCommandList（CPU 侧）
  2. 解析成 RenderScene
  3. 遍历场景树，对每个可见命令：
     a. 生成 2 个三角形（6 个顶点）→ vertex_buffer_
     b. 设置像素着色器参数 → constant_buffer_
     c. 选择混合模式 → blend_state_
     d. 调用 Draw(6, 0) → GPU 画出来
```
