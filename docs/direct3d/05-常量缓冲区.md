# 05 — 常量缓冲区

> 对应渲染器概念：`constant_buffer_`

## 问题

你的着色器需要知道一些"每帧都在变"的参数：
- 窗口宽度和高度（坐标转换要用）
- DPI 缩放比
- 当前是否启用了纹理
- 当前绘制的透明度

你可以把这些硬编码到着色器里，但每次窗口大小变了都要重新编译着色器——显然不行。

常量缓冲区就是解决这个问题的：**CPU 往里面写数据，GPU 在着色器里读**。

## 创建

```cpp
struct ConstantBufferData {
    float target_width;      // 窗口宽度（像素）
    float target_height;     // 窗口高度（像素）
    float dpi_scale;         // DPI 缩放
    int   enable_texture;    // 是否启用纹理
    float padding[3];        // 对齐到 16 字节
};

D3D11_BUFFER_DESC bd = {};
bd.ByteWidth      = sizeof(ConstantBufferData);
bd.BindFlags      = D3D11_BIND_CONSTANT_BUFFER;
bd.Usage          = D3D11_USAGE_DYNAMIC;        // CPU 每帧写入
bd.CPUAccessFlags = D3D11_CPU_ACCESS_WRITE;     // CPU 可写

device->CreateBuffer(&bd, nullptr, &constant_buffer_);
```

**关键点**：
- `Usage = DYNAMIC`：告诉 D3D11"这个缓冲区每帧都在变"
- 大小必须是 **16 字节的倍数**（D3D11 硬件要求）

## 更新

每帧渲染之前，把新的参数写入常量缓冲区：

```cpp
ConstantBufferData data;
data.target_width   = client_width;
data.target_height  = client_height;
data.dpi_scale      = dpi_;
data.enable_texture = has_texture ? 1 : 0;

D3D11_MAPPED_SUBRESOURCE mapped;
context->Map(constant_buffer_, 0, D3D11_MAP_WRITE_DISCARD, 0, &mapped);
memcpy(mapped.pData, &data, sizeof(data));
context->Unmap(constant_buffer_, 0);
```

`Map`/`Unmap` 是 CPU 和 GPU 之间传递数据的标准方式。`MAP_WRITE_DISCARD` 告诉 D3D11"我要覆盖整个缓冲区"——如果 GPU 还在读旧数据，D3D11 会给 CPU 一块新内存，不阻塞。

## 在着色器中使用

```hlsl
cbuffer Constants : register(b0) {
    float2 target_size;     // 窗口尺寸
    float  dpi_scale;       // DPI 缩放
    int    enable_texture;  // 纹理开关
};

float4 main(VSInput input) : SV_Position {
    // 把坐标从 DIP 转成 NDC（标准化设备坐标）
    float2 ndc = input.pos / target_size * 2.0f - 1.0f;
    return float4(ndc.x, -ndc.y, 0.0f, 1.0f);
}
```

`register(b0)` 表示"绑定到常量缓冲区槽 0"。CPU 侧的绑定也必须对应槽 0：

```cpp
context->VSSetConstantBuffers(0, 1, &constant_buffer_);
context->PSSetConstantBuffers(0, 1, &constant_buffer_);
```

**注意**：常量缓冲区对顶点着色器和像素着色器是**共享**的——同一个缓冲区可以同时绑定到两个阶段。

## 在你的渲染器中

你的渲染器每帧更新常量缓冲区一次：

```
prepare_ui_frame()
  → 计算 layout_constraints / target_pixel_size
  → 更新 constant_buffer_
  → 后续所有 Draw Call 都使用这个常量缓冲区的参数
```

你的 `constant_buffer_` 存储的内容：

| 字段 | 用途 |
|------|------|
| `target_width` / `target_height` | 窗口像素尺寸，用于坐标转换 |
| `textured` | 当前 Draw 是否使用纹理 |
| `texture_mode` | 纹理采样模式（alpha/次像素等） |

## 与顶点缓冲区的区别

```
顶点缓冲区                 常量缓冲区
─────────                  ─────────
存储顶点数据               存储全局参数
每帧可能几千个顶点          每帧几十个字节
GPU 只读（DEFAULT）        CPU 每帧写入（DYNAMIC）
绑定到 IA 阶段             绑定到 VS/PS 阶段
```
