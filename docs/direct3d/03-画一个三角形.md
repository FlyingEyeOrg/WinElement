# 03 — 画一个三角形

> 对应渲染器概念：`vertex_buffer_`、`vertex_shader_`、`pixel_shader_`、`input_layout_`、`Draw()`

这是 D3D11 编程的"Hello World"。理解了它，你就理解了 GPU 工作的基本模式。

## 完整流程

```
定义顶点数据 → 创建顶点缓冲 → 编译着色器 → 创建输入布局 → 设置管线 → Draw
```

## 第一步：定义顶点

画一个三角形需要 3 个顶点。每个顶点包含位置和颜色：

```cpp
struct Vertex {
    float x, y;      // 位置：2D 坐标
    float r, g, b;   // 颜色：RGB
};

Vertex triangle[] = {
    {  0.0f,  0.5f, 1.0f, 0.0f, 0.0f },  // 顶部: 红
    {  0.5f, -0.5f, 0.0f, 1.0f, 0.0f },  // 右下: 绿
    { -0.5f, -0.5f, 0.0f, 0.0f, 1.0f },  // 左下: 蓝
};
```

坐标是**标准化设备坐标（NDC）**：`(-1,-1)` 是左下角，`(1,1)` 是右上角。

## 第二步：创建顶点缓冲区

CPU 内存里的顶点数据要上传到 GPU 显存才能被 GPU 读取：

```cpp
D3D11_BUFFER_DESC bd = {};
bd.ByteWidth  = sizeof(triangle);     // 数据大小
bd.BindFlags  = D3D11_BIND_VERTEX_BUFFER;  // 这是顶点缓冲
bd.Usage      = D3D11_USAGE_DEFAULT;  // GPU 只读

D3D11_SUBRESOURCE_DATA init = {};
init.pSysMem = triangle;              // CPU 内存中的数据

ID3D11Buffer* vertex_buffer;
device->CreateBuffer(&bd, &init, &vertex_buffer);
```

`CreateBuffer` 做了两件事：
1. 在 GPU 显存中分配 `sizeof(triangle)` 字节
2. 把 CPU 内存的 `triangle` 数据拷贝过去

**对应你的渲染器**：`d3d11_display_list_renderer.hpp` 中的 `vertex_buffer_`。

## 第三步：编写着色器（HLSL）

HLSL 是 DirectX 的着色器语言，类似 C++ 但跑在 GPU 上。

**顶点着色器**——每个顶点执行一次：

```hlsl
struct VSInput {
    float2 pos : POSITION;    // 输入的坐标
    float3 color : COLOR;     // 输入的颜色
};
struct VSOutput {
    float4 pos : SV_Position; // 输出的位置（系统语义）
    float3 color : COLOR;     // 传给像素着色器
};

VSOutput main(VSInput input) {
    VSOutput output;
    output.pos = float4(input.pos, 0.0f, 1.0f);  // 直接透传
    output.color = input.color;
    return output;
}
```

**像素着色器**——每个像素执行一次：

```hlsl
float4 main(VSInput input) : SV_Target {
    return float4(input.color, 1.0f);  // 直接输出顶点的颜色
}
```

GPU 在顶点着色器和像素着色器之间会自动做**插值**：

```
三个顶点：红(0,1,0)  绿(0,1,0)  蓝(0,0,1)
     │
     ▼  GPU 自动插值
     │
三角形中间的一个像素：经过插值变成渐变混合色
```

编译着色器：

```cpp
ID3DBlob* vs_blob = nullptr;
D3DCompile(vertex_shader_source, strlen(...), 
           nullptr, nullptr, nullptr, 
           "main", "vs_5_0", 0, 0, &vs_blob, nullptr);

ID3D11VertexShader* vertex_shader;
device->CreateVertexShader(vs_blob->GetBufferPointer(), 
                           vs_blob->GetBufferSize(), 
                           nullptr, &vertex_shader);
```

**注意**：`D3DCompile` 需要链接 `d3dcompiler.lib`。

## 第四步：创建输入布局

输入布局告诉 GPU"顶点数据在内存里是怎么排列的"：

```cpp
D3D11_INPUT_ELEMENT_DESC layout[] = {
    // 语义名    索引  格式                 偏移  每个顶点的数据
    { "POSITION", 0, DXGI_FORMAT_R32G32_FLOAT,   0, 0,  PER_VERTEX_DATA, 0 },
    { "COLOR",    0, DXGI_FORMAT_R32G32B32_FLOAT, 0, 8,  PER_VERTEX_DATA, 0 },
};
device->CreateInputLayout(layout, 2, vs_blob->GetBufferPointer(), 
                          vs_blob->GetBufferSize(), &input_layout);
```

这个结构的意思是：

```
顶点 (20 字节)
├── 偏移 0:  x (float, 4 字节)
├── 偏移 4:  y (float, 4 字节)     ← POSITION
├── 偏移 8:  r (float, 4 字节)
├── 偏移 12: g (float, 4 字节)     ← COLOR
└── 偏移 16: b (float, 4 字节)
```

**对应你的渲染器**：`input_layout_` 在 `d3d11_display_list_renderer.hpp` 中。

## 第五步：设置管线状态并 Draw

```cpp
// 1. 清屏
context->ClearRenderTargetView(render_target_view, clear_color);
context->OMSetRenderTargets(1, &render_target_view, nullptr);

// 2. 设置视口
D3D11_VIEWPORT vp = { 0, 0, 800, 600, 0, 1 };
context->RSSetViewports(1, &vp);

// 3. 绑定输入布局 + 图元拓扑
context->IASetInputLayout(input_layout);
context->IASetPrimitiveTopology(D3D11_PRIMITIVE_TOPOLOGY_TRIANGLELIST);

// 4. 绑定顶点缓冲区
UINT stride = sizeof(Vertex);  // 每个顶点 20 字节
UINT offset = 0;
context->IASetVertexBuffers(0, 1, &vertex_buffer, &stride, &offset);

// 5. 绑定着色器
context->VSSetShader(vertex_shader, nullptr, 0);
context->PSSetShader(pixel_shader, nullptr, 0);

// 6. 画！
context->Draw(3, 0);  // 3 个顶点，从偏移 0 开始

// 7. 显示
swapchain->Present(1, 0);
```

`Draw(3, 0)` 这一行，是整条管线的终点：

```
Draw(3, 0)
  → GPU 读取 vertex_buffer_ 的前 3 个顶点
  → 顶点的位置数据传给 Vertex Shader
  → Vertex Shader 输出坐标
  → Rasterizer 把三角形转成像素 (光栅化)
  → 每个像素执行 Pixel Shader（颜色取插值结果）
  → Output Merger 写回渲染目标
  → Present 显示到屏幕
```

## 你的框架怎么用这些

你的框架不画三角形，它画的是 UI 控件。但原理一样：

| UI 控件 | 顶点的意义 |
|---------|-----------|
| 一个矩形 | 2 个三角形 = 6 个顶点 |
| 圆角矩形 | 更多三角形 = GPU 细分 |
| 一行文字 | 每个字形是一个四边形 = 2 个三角形 |
| 一张图片 | 一个四边形带纹理坐标 = 2 个三角形 |

你的 `vertex_buffer_` 每帧被填充成千上万个顶点，然后 `Draw` 被调用几十到几百次，每次画出 UI 中的一个可见元素。

## 总结

```
数据准备                    GPU 管线
─────────                  ────────
顶点数组                    Input Assembler → 读顶点
  │                              │
CreateBuffer              Vertex Shader → 坐标转换
  │                              │
编译 HLSL                  Rasterizer → 三角形→像素
  │                              │
CreateInputLayout          Pixel Shader → 算颜色
  │                              │
绑定管线状态                Output Merger → 混合写入
  │                              │
Draw(3, 0) ───────────►   Present → 显示到屏幕
```
