# 04 — 渲染状态

> 对应渲染器概念：`blend_state_*`、`stencil_*_state_`、`rasterizer_state_`、`sampler_state_`

画三角形只需要最基本的管线设置。画真实的 UI 还需要控制 GPU 如何处理颜色混合、裁剪、纹理采样等问题。这就是渲染状态做的事。

## 四种状态

```cpp
// d3d11_display_list_renderer.hpp
ID3D11BlendState*             blend_state_;                // 透明度混合
ID3D11BlendState*             subpixel_text_blend_state_;  // 文本亚像素混合
ID3D11BlendState*             stencil_mask_blend_state_;   // 模板遮罩混合

ID3D11RasterizerState*        rasterizer_state_;           // 光栅化

ID3D11SamplerState*           sampler_state_;              // 纹理采样

ID3D11DepthStencilState*      stencil_disabled_state_;     // 模板：关闭
ID3D11DepthStencilState*      stencil_test_state_;         // 模板：测试
ID3D11DepthStencilState*      stencil_increment_state_;    // 模板：递增
ID3D11DepthStencilState*      stencil_decrement_state_;    // 模板：递减
```

## 混合状态（Blend State）

控制新画的像素如何与屏幕上已有的像素混合。

**标准 alpha 混合公式**：
```
最终颜色 = 源颜色 × 源透明度 + 目标颜色 × (1 - 源透明度)
         = new_rgba * src_alpha + old_rgba * (1 - src_alpha)
```

创建混合状态：

```cpp
D3D11_BLEND_DESC bd = {};
bd.RenderTarget[0].BlendEnable    = TRUE;
bd.RenderTarget[0].SrcBlend       = D3D11_BLEND_SRC_ALPHA;        // 源 = alpha
bd.RenderTarget[0].DestBlend      = D3D11_BLEND_INV_SRC_ALPHA;    // 目标 = 1-alpha
bd.RenderTarget[0].BlendOp        = D3D11_BLEND_OP_ADD;            // 相加
bd.RenderTarget[0].RenderTargetWriteMask = D3D11_COLOR_WRITE_ENABLE_ALL;
device->CreateBlendState(&bd, &blend_state_);
```

**三种混合状态的用途**：

| 状态 | 场景 | 区别 |
|------|------|------|
| `blend_state_` | 普通 UI 元素（矩形、几何） | 标准 alpha 混合 |
| `subpixel_text_blend_state_` | ClearType 文字 | 每个子像素独立混合，文字更锐利 |
| `stencil_mask_blend_state_` | 模板遮罩 | 特殊混合模式，只画到模板标记区域 |

## 光栅化状态（Rasterizer State）

控制三角形如何被转换成像素。

```cpp
D3D11_RASTERIZER_DESC rd = {};
rd.FillMode              = D3D11_FILL_SOLID;    // 填充模式（实心/线框）
rd.CullMode              = D3D11_CULL_NONE;     // 不裁剪背面（UI 不需要）
rd.ScissorEnable         = TRUE;                // 启用裁剪矩形
device->CreateRasterizerState(&rd, &rasterizer_state_);
```

**关键设置**：
- `CullMode = D3D11_CULL_NONE`：游戏要决定哪些面是背面不画，UI 不需要，全都画
- `ScissorEnable = TRUE`：启用矩形裁剪，配合 `RSSetScissorRects` 使用

**你的脏区域裁剪就是通过 Scissor Rect 实现的**：

```cpp
// 对于每个脏区域矩形，设置裁剪矩形
context->RSSetScissorRects(1, &dirty_rect);
// 然后画——GPU 会自动裁掉脏区域外的像素
context->Draw(6, 0);
```

这比在像素着色器里判断效率高得多——硬件级别的裁剪。

## 采样器状态（Sampler State）

控制纹理采样时的行为。

```cpp
D3D11_SAMPLER_DESC sd = {};
sd.Filter        = D3D11_FILTER_MIN_MAG_LINEAR_MIP_POINT;  // 线性插值
sd.AddressU      = D3D11_TEXTURE_ADDRESS_CLAMP;             // 钳位
sd.AddressV      = D3D11_TEXTURE_ADDRESS_CLAMP;
device->CreateSamplerState(&sd, &sampler_state_);
```

UI 纹理采样的关键点：**CLAMP 模式**。游戏常用 WRAP（重复纹理），UI 用 CLAMP（超出纹理边界时取边缘像素颜色），不然图集边缘会出现其他字形的颜色。

## 深度/模板状态（Depth/Stencil State）

D3D11 把深度测试和模板测试合并在同一个状态下。UI 框架不用深度测试（不画 3D），但**大量使用模板测试**来实现裁剪。

```cpp
// 模板：关闭
D3D11_DEPTH_STENCIL_DESC dsd = {};
dsd.DepthEnable = FALSE;
dsd.StencilEnable = FALSE;
device->CreateDepthStencilState(&dsd, &stencil_disabled_state_);

// 模板：测试（只画模板值 == 1 的像素）
dsd.StencilEnable = TRUE;
dsd.StencilReadMask  = 0xFF;
dsd.StencilWriteMask = 0xFF;
dsd.FrontFace.StencilFunc        = D3D11_COMPARISON_EQUAL;
dsd.FrontFace.StencilPassOp      = D3D11_STENCIL_OP_KEEP;
dsd.FrontFace.StencilFailOp      = D3D11_STENCIL_OP_KEEP;
dsd.FrontFace.StencilDepthFailOp = D3D11_STENCIL_OP_KEEP;
device->CreateDepthStencilState(&dsd, &stencil_test_state_);
```

这四种模板状态在下一章详细展开。

## 在渲染管线中的位置

这些状态是在 Draw Call 之前设置的，而且**频繁切换状态是性能杀手**：

```
高效的绘制顺序：
  SetBlendState(Alpha)
  SetRasterizerState(ScissorEnabled)
  Draw(矩形1)
  Draw(矩形2)       ← 状态没变，GPU 不用切
  Draw(矩形3)
  SetBlendState(SubpixelText)  ← 切一次状态
  Draw(文字1)
  Draw(文字2)

低效的绘制顺序：
  SetBlendState(Alpha)
  Draw(矩形1)
  SetBlendState(SubpixelText)   ← 切
  Draw(文字1)
  SetBlendState(Alpha)          ← 又切回来
  Draw(矩形2)
```

你的 `RenderDrawBatch` 就是为了把相同状态的命令合并，减少状态切换。

## 对应你的渲染器

| 状态 | 在 d3d11_display_list_renderer 中的位置 |
|------|---------------------------------------|
| 创建所有状态 | `create_pipeline()` 函数中 |
| 应用混合状态 | `RecorderState::bind_blend_state()` |
| 应用裁剪矩形 | `apply_current_scissor()` |
| 应用模板状态 | `RecorderState::set_stencil_mode()` |
| 应用采样器 | `draw_vertices_now()` 中绑定 |
